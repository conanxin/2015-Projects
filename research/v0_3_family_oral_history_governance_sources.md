# V0.3 · Family Oral History Governance — Source Survey

> Survey prepared 2026-07-11 for the **2015 Projects Reborn / 个人知识与记忆实验室**
> V0.3 family-oral-history governance work package. No real interview, no real
> narrator, no real recording was conducted for this task.

This document is **not** a substitute for legal advice. The interview consent
record, recording permission, AI processing options, and publication approval
templates in [`docs/oral-history/templates/`](../docs/oral-history/templates/)
are **operational templates** for a single-operator project. They are not a
jurisdiction-specific contract. Local counsel must review any document that
will be signed by a real narrator.

## 1. Why this survey exists

V0.2B shipped the *product model* for a public oral-history story page using a
public archive (NPS Ellis Island, Isadore Samet, EI-87, 41.8 s). V0.3 prepares
for the next step, which must be **ethical and operational, not technical**:
interviewing a real narrator from outside a public archive. This survey maps
the publicly available best-practice material that defines what "ethical and
operational" means in 2026.

## 4. Three-tiered principle set (adopted)

This project does **not** adopt a full deed-of-gift, nor does it copy any
single institution's consent wording. It builds a project-specific work package
that respects the principles below at three levels:

### A. Ethics & best-practice principles (in this repo, applied without counsel)
- Narrator is a participant, not a data source (OHA #4, #5).
- Consent is **ongoing** (OHS #8, OHA #1).
- The post-interview plan starts *before* the narrator is approached (OHA #4).
- Embargo and controlled access are defaults, not exceptions (OHA #3,
  UCSB DLS #18).
- AI-assisted processing is **separately** consented (OHA #6).
- Voice cloning is treated as forbidden without explicit, separate,
  revocable consent (OHA #6; OHA #5).
- Sensitive material categories (minors, family disputes, finances, health,
  religion, politics, etc.) are recorded with private reviewers and a
  redaction workflow before any public release (OHA #3, #5; Penn State #11).
- Transcript conventions follow a manual-of-style approach (Baylor #13) but
  with a deliberately minimal set of markers (only what this project
  actually needs).

### B. Operational & technical rules (in this repo)
- A four-tier data classification (Level 0 raw private, Level 1 restricted
  working, Level 2 approved release, Level 3 public published).
- Project identifiers in the form `OH-YYYY-NNN`; session identifiers
  `OH-YYYY-NNN-S##`; no real names, phone numbers, addresses, or IDs in
  filenames.
- A schema-validated `interview-project`, `participant-consent`,
  `interview-session`, `publication-decision`, and `withdrawal-request`
  JSON shape.
- A local validator that ensures (1) every required template exists, (2)
  every schema is valid JSON-Schema shaped, (3) no real personal
  information leaks into the public docs, and (4) the public docs reference
  the right level for each artifact.

### C. Documents that are NOT adopted as legal contracts
- The V0.3 consent record, recording permission, AI processing options,
  publication approval, and withdrawal request **ARE** project-internal
  operational templates. They are **NOT** jurisdiction-specific legal
  contracts. They should be reviewed by counsel before being signed if they
  are intended to have legal force in the narrator's jurisdiction.

## 5. Six concepts this project enforces via docs/oral-history/

1. **Modular consent, not blanket consent** — every consent record has
   checkboxes for media, processing, identity, content form, and timing
   (OHS #8; OHA #2; UCSB DLS #18).
2. **Default-private publishing** — anything not explicitly approved stays
   inside `oral-history-private/` or `oral-history-working/` (OHA #3,
   UCSB DLS #18; Heritage #20).
3. **Public Git never holds Level 0 or Level 1 material** — enforced by
   both `.gitignore` narrow rules and a final check in
   `scripts/validate-oral-history-governance.mjs`.
4. **AI processing options are explicitly opt-in per option** — voice
   cloning, voice-as-data for cloning, model fine-tuning, training
   inclusion, and AI-as-narrator are **default-prohibited** (OHA #6).
5. **Three reviewer tiers (Review A: facts & identity; Review B:
   publication boundary; Review C: final page)** — explicit narrator
   authority over personal content, editor responsibility for fact
   background (OHA #1, #2; Nunn Center #12).
6. **Six publication gates, none skippable** — Gate 0 planned →
   Gate 1 recorded → Gate 2 working draft → Gate 3 narrator review →
   Gate 4 public candidate → Gate 5 published → ongoing maintenance
   (LOC VHP #10; OHS #7; Penn State #11).

## 6. What is intentionally NOT in this work package

- A pre-filled consent form for any specific narrator.
- A story page about any real family member.
- An audio or video file of a real person.
- An IRC-bot, a wiki, an OHMS server, an Omeka instance, or any other
  service back-end.
- Database files, CSV dumps, or sqlite snapshots.
- Jurisdiction-specific legal language.
