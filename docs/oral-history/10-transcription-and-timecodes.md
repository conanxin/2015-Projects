# 10 · 转录与时间码政策

> 出处：[Baylor #13] Style Manual; [AudiAnnotate #19] Audio Annotation Standards; [OHA #1] Best Practices; [OHA #6] AI Annual Meeting panel。
>
> 本文与 `templates/redaction-log.md`、`schemas/interview-session.schema.json`
> 配套。

## 10.1 两种转录：原始 + 编辑

| 类型 | 文件 | 内容 | 谁能看 |
|---|---|---|---|
| 原始转录 | `*_transcript-raw.md` | 听不清使用统一标记，不做任何语言修订 | Level 0 / Level 1（操作者） |
| 编辑转录 | `*_transcript-edited_v##.md` | 经过语言修订的版本，可读性优先 | Level 1 / Level 2（依同意书） |

两类**必须分开**；原始转录不被任何后续工作覆盖。

## 10.2 转录标记（统一集）

只使用以下 6 类标记，绝不引入新标记，避免人工复听混乱：

| 标记 | 含义 |
|---|---|
| `[听不清 hh:mm:ss]` | 该秒附近无法听清的内容 |
| `[人名待核实]` | 人名拼写 / 拼写尚未核实 |
| `[日期约为 X]` | 叙述者明确说了"大概是 X"，不强行锁定 |
| `[讲述者笑]` | 说话中有笑声，但不是说出来的话语 |
| `[停顿]` | 长达 5 秒以上的沉默 |
| `[经讲述者要求删除]` | 必须在 Level 1 受限工作稿中，并打码，不得公开展示敏感上下文 |

`[经讲述者要求删除]` 这种标记**只出现在 Level 1 工作稿**，**绝不**
出现在公开 Level 2 / Level 3 页面。

## 10.3 引用规则（重要）

- **公开页面摘要不使用引号**（它是"编辑摘要"，不是"逐字引语"）。
- **只有逐字原话才使用引号** —— 并明确加 `[逐字, mm:ss]` 时间码锚点。
- **AI 改写内容绝不标成原话** —— 这是 [OHA #6] 的核心。
- **人名、地名、年份必须核实** —— 任何没核实之前使用占位符：
  `[人名待核实]` / `[地点待核实]` / `[年份待核实]`。

## 10.4 时间码规则

- **公开页面默认使用章节级时间码**（30 秒到 5 分钟一章），不要求逐词同步。
- 每个章节保留 1–12 词的英文短摘录 + 同长度的中文短译 + 1–2 句中文编辑摘要。
- 短摘录**只起锚点作用**；不展示整句。
- AI 转录必须人工复听，并在 `redaction-log.md` 中注明哪些段落被人工修正。

## 10.5 不进入公共 Git 的内容

下面这些**绝不出现在公共 Git 中**：

- 任何未经审核的完整转录；
- 任何包含敏感第三方信息的摘录；
- 任何包含人名 / 生日 / 家庭住址 / 健康信息的摘录；
- 任何 `AI 自动转录 - 未复听` 状态的转录稿；
- `[经讲述者要求删除]` 等敏感标记；
- `withdrawal-log.md`、`redaction-log.md` 等流程记录。

## 10.6 与 schemas 的关系

- `schemas/interview-session.schema.json` 中保留字段 `transcriptHash` 与
  `timecodeHash`（仅 SHA-256，不含内容），用于在 Level 2 校验 Level 1
  转录是否仍存在；不外泄内容。
- `schemas/publication-decision.schema.json` 中保留 `approvedSegments`，
  仅记录被批准的章节编号。

## 10.8 转录员与编辑员的角色区分

| 角色 | 任务 |
|---|---|
| 转录员（操作者本人） | 听录音 → 出原始转录 |
| 编辑员 | 校正拼写 / 语法 → 出编辑转录 |
| 审核员（讲述者） | 检查事实、同意边界、口语化"失误"是否保留 |

**这三者不能由同一人同时担任。** 单人项目情况下，操作者轮流切换角色并
记录每次切换日期。
