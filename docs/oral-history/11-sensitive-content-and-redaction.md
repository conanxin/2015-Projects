# 11 · 敏感内容与遮蔽

> 出处：[OHA #5] Statement on Ethics; [OHS #7] Legal and Ethical; [Penn State #11] Best Practices; [UCSB DLS #18] Embargo; [Heritage #20] Oral history good practice guidance。
>
> 本文把 14 类敏感内容做了硬性默认设置。每类都说明：默认值、是否
> 允许进入工作稿、公开前需要谁批准、推荐的脱敏方法、是否要完全排除。

## 11.1 14 类敏感内容

### 类别一览

| 类别 | 默认 | 进 Level 1 工作稿? | 公开前谁批 | 推荐脱敏 | 完全排除? |
|---|---|:---:|---|---|:---:|
| 未成年人面孔 / 声音 | **私密** | 否（除非本人同意 + 监护人同意） | 监护人 + 讲述者 | 化名 + 不放原声 / 原图 | 常需 |
| 精确家庭住址 | **私密** | 否 | 讲述者 | 改为地区级（"北方某城市"） | 常需 |
| 手机号 / 邮箱 | **私密** | 不存 | – | 直接删除 | 必须 |
| 身份证件号码 | **私密** | 不存 | – | 直接删除 | 必须 |
| 健康 / 医疗 | 私密 | 仅在同意书内说明时 | 讲述者 + 医生（如必要） | 化名 + 概述 / 时间模糊 | 视情况 |
| 财务状况 | 私密 | 概述 | 讲述者 | 改为范围级（"大企业"） | 视情况 |
| 家庭矛盾 | 私密 | 仅在同意书内说明时 | 当事双方分别签字 | 改为"存在分歧"叙述 | 视情况 |
| 政治观点 | 私密 | 概述 | 讲述者 | 用时间区间而非具体日期 | 视情况 |
| 宗教信仰 | 私密 | 概述 | 讲述者 | 改为"该家庭信仰某宗教" | 视情况 |
| 犯罪 / 指控 | **私密** | 不存（除非已公开案件） | 律师 | 永远不录 | 必须 |
| 第三方未经证实的评价 | **私密** | 否 | 该第三方 | 永远不录 | 必须 |
| 性 / 亲密关系 | 私密 | 仅在同意书内说明时 | 讲述者 | 完全删除或泛化为"成家有孩子" | 常需 |
| 创伤经历 | **私密** | 仅 Level 0，封闭保存 | 讲述者 | 时间模糊 + 概述 | 必须 |
| 尚未公开的死讯 / 收养 / 亲缘 | **私密** | 不存 | 家属全部 | 完全不提 | 必须 |
| 工作单位内部信息 | 私密 | 概述 | 讲述者 + 雇主（如必要） | 时间脱敏 + 内容脱敏 | 必须 |

## 11.2 敏感不等于"只要删掉姓名就行"

**关键警告**：组合信息仍可识别个人。"1990 年代 + 大公司 + 广东 + 大专 + 一个孩子" 在很多
小型社区就足以识别。`redaction-log.md` 必须记录"组合可识别性"，**不只**记录删除的字段。

## 11.3 实际脱敏方法（5 类）

1. **删除**：彻底不提。
2. **化名**：用同音 / 同义指代，但要避免使用真人姓名近音或名人姓名。
3. **时间模糊**：从"2014 年 3 月"改为"2014 年初"或"2014 年春"。
4. **空间模糊**：从"广州市天河区"改为"珠三角某城市"或"南方某城"。
5. **概括化**：用一段话概述，而不是逐字引用。

## 11.4 完全排除的清单（绝不写入任何稿件）

- 任何**未公开**的死亡 / 收养信息（即使讲述者本人同意，仍需家属全体
  签字）；
- 任何**犯罪细节 / 指控**（除非已经公开判决）；
- **任何一方的**手机号 / 邮箱 / 身份证号 / 护照号 / 银行账号；
- 任何**第三方未成年儿童**的真实姓名 / 学校 / 班级 / 好友名字；
- 任何**正在进行的法律程序**细节。

## 11.5 第三方独立同意

- 即便讲述者本人同意，第三方有独立同意权利。
- 操作员必须**逐人**而非"批量"获得第三方同意。
- 任何第三方拒绝的部分**必须**删除或概括。
- 这条规则**优先于**讲述者的"我同意"——第三方否认权不可被覆盖。

## 11.6 公开页面前的"敏感复审"

每次提交公开页面前，下列复审项在 `templates/public-release-checklist.md`
中被机械检查：

- [ ] 没有身份证号；
- [ ] 没有手机号 / 邮箱；
- [ ] 没有完整地址（精确到门牌）；
- [ ] 没有未成年人面孔 / 真实姓名（除非同意 + 监护人同意）；
- [ ] 没有尚未公开的死讯 / 收养；
- [ ] 没有犯罪细节；
- [ ] 没有未脱敏的第三方姓名 / 第三方负面评价；
- [ ] 没有"删除姓名但留下其它可识别组合"的情况；
- [ ] 没有超过同意书范围的资料；
- [ ] 没有 AI 自动生成、被冒充原话的内容；
- [ ] 没有 Level 0 文件被无意间包含在 commit 中。

## 11.7 后悔处理（"我之前没意识到这会泄露"）

- 讲述者在公开后**任何时间**都可发起修改；
- 操作员在收到请求的 14 天内完成修改；
- 行为按 `04-withdrawal-and-change-requests.md` 处理；
- 这条规则**独立于**同意书的原始范围——一旦发起来，必须响应。
