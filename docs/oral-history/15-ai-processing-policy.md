# 15 · AI 处理政策

> 出处：[OHA #6] AI Annual Meeting 2025 panel; [OHA #5] Statement on Ethics; [OHA #1] Best Practices。
>
> 本文针对本项目实际使用的工具——Hermes（本地 agent）、DocuMuse
>（内容类型识别）、`ffmpeg` / `ffprobe` / `whisper-cpp`（本地推理）、
> 以及可能的云端模型——给出"默认允许 / 单独同意 / 默认禁止"三层
> 规则。

## 15.1 默认允许（在讲述者同意的前提下，操作员可直接使用）

- **本地离线音频分析**（ffmpeg / ffprobe / 波形分析工具）；
- **本地离线转录**（whisper-cpp 在本机运行，不上传任何字节）；
- **本地时间码辅助**（脚本生成章节起止时间，操作员最终确认）；
- **不保存到第三方服务的本地摘要辅助**（操作员内部的 Markdown 笔记）；
- **本地检索 / 链接检查**（如 `site/scripts/` 下的本地脚本）；
- **本地 schema 验证**（如 `site/scripts/validate-story-data.mjs`）。

这些都属于"操作员日常工作"，不涉及"将讲述者资料传给第三方"。

## 15.2 需要单独明确同意（默认禁用）

| 项 | 默认 | 单独同意要求 |
|---|---|---|
| 上传音频到云端语音识别 | ❌ | 须**逐项勾选** + 二次确认 |
| 上传完整转录到云端 LLM | ❌ | 须**逐项勾选** + 列明厂商 |
| 使用云端翻译 | ❌ | 同上 |
| 使用云端摘要 | ❌ | 同上 |
| 把资料放入联网知识库 | ❌ | 不仅需勾选，还需**预先**签 NDA |
| 跨项目复用 | ❌ | 须项目双方签字 + 重新声明 |

每项勾选都需要在 `templates/ai-processing-options.md` 单独列出
**输入范围 / 输出范围 / 厂商 / 保留时间**。未填字段视为"未同意"。

## 15.3 默认禁止（永远不经同意就能开启）

| 禁止项 | 仅在【明确勾选 + 二次确认】时才能做 |
|---|:---:|
| **声音克隆**（voice cloning）：训练或 fine-tune 模型以模仿讲述者声音 | ☐ —— 通常**不**勾 |
| **以讲述者声音生成新话语**：让 AI 说出从未说过的话 | ☐ —— 通常**不**勾 |
| **模型训练或微调**：用讲述者音频 / 转录训练新模型 | ☐ —— 通常**不**勾 |
| **发布 AI 伪造的讲述者照片**：用生成式图像伪造讲述者 | ☐ —— 通常**不**勾 |
| **把未经批准的内容加入公共聊天机器人**：让陌生人通过 AI 询问 | ☐ —— 通常**不**勾 |
| **让 AI 自动决定哪些敏感内容可以公开** | ☐ —— 通常**不**勾 |
| **将 AI 输出直接视为事实** | ☐ —— 通常**不**勾 |
| **将 AI 改写内容标成原话** | ☐ —— 通常**不**勾 |

这一层**绝不**仅靠"项目方做出合理判断"来打开；只有讲述者**逐项**
明确勾选 + 二次确认才能开启。

## 15.4 每次 AI 处理的最少记录

`templates/ai-processing-options.md` 的每一次使用都应有记录条目：

```
AI 处理记录 vNN (date: YYYY-MM-DD)
- 工具: whisper-cpp / Claude API / Otter / ffmpeg / ...
- 本地 / 云端: local / cloud
- 输入范围: 音频文件路径 / 转录文件路径 / 章节描述
- 输出范围: 文本 / JSON / 时间码 / 摘要 / 翻译 / 嵌入
- 是否保留: 是 / 否（保留则路径；不保留则"未保留"）
- 是否经人工审核: 是（签名） / 否
- 对应 consent 选项: A-某 / B-某 / ...
```

## 15.5 操作员的内部 guard

- Hermes 在被要求"使用云端模型处理访谈"时，**必须**先检查
  `templates/ai-processing-options.md` 中是否含有效记录；
- 没有有效记录 → Hermes 拒绝执行该操作并解释原因。
- 这条 guard 与脚本 `scripts/validate-oral-history-governance.mjs` 中
  对模板的检查共同生效。

## 15.6 与 docs/oral-history/ 其它部分的关系

- 本文是 03（处理同意）+ 04（撤回）+ 11（敏感）+ 13（gate）的 AI 视角总结。
- 任何 AI 处理超出本文档允许范围，等同于"无同意"，等同于"违反治理"。
- 任何 AI 输出被冒充原话，等同于违反 12（讲述者审核）。

## 15.7 故事的"无 AI 路径"

讲述者**完全**可以选择"全程人工"的路径：

- 不上传任何音频 / 转录到云端；
- 只用本地 Whisper / ffmpeg；
- 章节时间码由操作员人工复听生成；
- 不使用任何 LLM 摘要或翻译；
- 公开页面只是事实陈述 + 短摘录 + 中文编辑摘要。

这条路径**完全不需要** § 15.2 的任何勾选，也不需要 § 15.4 的
记录条目。这是默认"环保"路径。

## 15.8 故事的"全 AI 路径"

讲述者**只在**全部勾选 §15.2 + 全部取消 §15.3 默认禁止时才允许：

- 不应**整体**勾选；
- 任何 §15.2 项一旦勾选就**立即生效**单次；下次再需重新勾选；
- 任何 §15.3 项**仅**在二次确认（单独邮件 / 视频 / 见证人）
  后才生效。

如果讲述者在 §15.2 多次勾选、§15.3 没有一次勾选，这就是
AI 辅助的合理边界。

## 15.9 AI 与隐私：组合风险

[OHA #6] 强调 AI 输出本身可能成为隐私泄漏载体：

- AI 摘要中的推断可能"超出讲述者原话"；
- AI 翻译中的同义词可能产生原文未授权的暗示；
- AI 嵌入向量可能被反向检索出原文片段。

因此：

- 公共页面**不**展示 AI 摘要原文；
- 公共页面**不**展示 AI 翻译结果（讲述者保留审核权）；
- 公共页面只展示经过操作者人工确认的原文 + 操作员撰写的中文编辑摘要。
